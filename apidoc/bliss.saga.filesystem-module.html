File/Directory Management

The file managment API provides the ability to interact with (local and remote) file systems via the two classes, filesystem.Directory and filesystem.File. The API provides a number of operations, which all behave similar to the common unix command line tools (cp, ls, rm etc).

Example:

 # get a directory handle
 dir = saga.filesystem.Directory("sftp://localhost/tmp/")

 # create a subdir
 dir.make_dir ("data/")

 # list contents of the directory
 files = dir.list ()

 # copy *.dat files into the subdir
 for f in files :
     if re.match ('^.*\.dat$', f) :
         dir.copy (f, "sftp://localhost/tmp/data/")

The above example covers most of the semantics of the filesystem package -- additional capabilities, such get_size() or move(), can be found in the individual class documentations.

Variables Details

Create

Allows a namespace entry to be created while opening it, if it does not already exist – if that flag is not given, the same open operation would cause a ’DoesNotExist’ exception. If the entry exists, the flag is ig- nored. This flag implies the ’Write’ flag.

Value:
8

Lock

Enforces a lock on the name space entry when it is opened. Locks are advisory in SAGA, semantic details for locking are defined in the description of the open() call.

Value:
32

CreateParents

An operation which would create a name space entry would normally fail if any path element in the targets name does not yet exist. If this flag is given, such an operation would not fail, but would imply that the missing path elements are created on the fly. This flag implies the ’Create’ flag.

Value:
64
