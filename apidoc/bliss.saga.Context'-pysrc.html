 18      '''Loosely defines a SAGA Context object as defined in GFD.90. 
 19   
 20      A security context is a description of a security token.  It is important to 
 21      understand that, in general, a context really just *describes* a token, but 
 22      that a context *is not* a token (*). For example, a context may point to 
 23      a X509 certificate -- but it will in general not hold the certificate 
 24      contents. 
 25   
 26      Context classes are used to inform the backends used by Bliss on what 
 27      security tokens are expected to be used.  By default, Bliss will be able to 
 28      pick up such tokens from their default location, but in some cases it might 
 29      be necessary to explicitly point to them - then use a L{Session} with 
 30      context instances to do so. 
 31   
 32      The usage example for contexts is below:: 
 33   
 34          # define an ssh context 
 35          c = saga.Context() 
 36          c.context_type = 'ssh' 
 37          c.user_cert = '$HOME/.ssh/special_id_rsa' 
 38          c.user_key = '$HOME/.ssh/special_id_rsa.pub' 
 39   
 40          # add the context to a session 
 41          s = saga.Session() 
 42          s.contexts.append(c) 
 43   
 44          # create a job service in this session -- that job service can now 
 45          # *only* use that ssh context.  
 46          j = saga.job.Service('ssh://remote.host.net/', s) 
 47   
 48   
 49      The L{Session} argument to the L{job.Service} constructor is fully optional 
 50      -- if left out, Bliss will use default session, which picks up some default 
 51      contexts as described above -- that will suffice for the majority of use 
 52      cases. 
 53   
 54      ---- 
 55   
 56      (*) The only exception to this rule is the 'UserPass' key, which is used to 
 57      hold plain-text passwords.  Use this key with care -- it is not good 
 58      practice to hard-code passwords in the code base, or in config files. 
 59      Also, be aware that the password may show up in log files, when debugging or 
 60      analyzing your application. 
 61   
 62      '''
