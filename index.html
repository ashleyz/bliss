Bliss

A Pragmatic SAGA Implementation for Python

  • Bliss (BLiss IS Saga) is a pragmatic, light-weight implementationof the OGF GFD.90 SAGA standard. Bliss is written 100% in Python and focuses on usability and simple deployment rather than on feature completeness.

    SAGA defines a high-level API for uniform access to the building-blocks of distributed computing: (1) job execution and managegment, (2) file and data management and (3) distributed coordination and orchestration. Bliss is a native Python implementation of this API.

    So why should I use Bliss?

    With Bliss you can write distributed application pipelines and workflows that are portable accross distributed systems. With Bliss you can finally say good bye to all those aweful, error-prone glue-code scripts that you use to handle compute and data provisioning and management (or avoid them in the first place).

    But wait - Bliss can also be a great tool for your research in computer science! You have great idea for an algorithm that does data/compute co-scheduling accross XSEDE and Open Science Grid machines? Use Bliss to encapsulate resource access details. This will give you more time to focus on the important things and less things to wory about.
