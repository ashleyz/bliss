  • What is Bliss?

    Bliss (BLiss IS Saga) is a pragmatic, light-weight implementation of the OGF GFD.90 SAGA standard. Bliss is written 100% in Python and focuses on usability and simple deployment rather than on feature completeness and strict standard obedience.

    The SAGA standard defines a high-level API for uniform acess to the building-blocks of distributed computing: (1) job execution and managegment, (2) data management and (3) distributed coordination and orchestration. Bliss is a native Python implementation of this API with flexible plug-in mechanism to support different distributed backends.

    Why should I use Bliss?

    Bliss allows you to write distributed applications and workflows that are portable accross distributed systems. With Bliss you can finally say goodbye to all those awful, error-prone glue-code scripts that you use to handle compute and data provisioning and management for your application.

    Bliss is also a great tool to carry out your distributed computing and computational science research: you have great idea for an algorithm that does data/compute co-scheduling in a heterogeneous resource environment? You want to test it in a real-life scenario? Bliss allows you to encapsulate heterogeneous resource access details and lets you focus on the exciting part of your work!
