 14      A Bliss session has the purpose of scoping the use of security credentials 
 15      for remote operations.  In other words, a session instance acts as 
 16      a container for security L{Context} instances -- Bliss objects (such as 
 17      L{job.Service} or L{filesystem.File}) created in that session will then use 
 18      exactly the security contexts from that session (and no others). 
 19   
 20      That way, the session serves two purposes:  (1) it helps Bliss to decide 
 21      which security mechanism should be used for what interaction, and (2) it 
 22      helps Bliss to find security credentials which would be difficult to pick up 
 23      automatically. 
 24       
 25      The use of a session is as follows: 
 26   
 27   
 28      Example:: 
 29   
 30   
 31          # define an ssh context 
 32          c = saga.Context() 
 33          c.type = 'ssh' 
 34          c.usercert = '$HOME/.ssh/special_id_rsa' 
 35          c.userkey = '$HOME/.ssh/special_id_rsa.pub' 
 36   
 37          # add it to a session 
 38          s = saga.Session 
 39          s.add_context(c) 
 40   
 41          # create a job service in this session -- that job service can now 
 42          # *only* use that ssh context.  
 43          j = saga.job.Service('ssh://remote.host.net/', s) 
 44   
 45   
 46      The session argument to the L{job.Service} constructor is fully optional --
