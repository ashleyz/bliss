Loosely represents a SAGA job description as defined in GFD.90

The job.Description class is essentially a table of key/value pairs (dict),
which describes both the resource (e.g. number of CPUs, working directory
etc.), and the application properties (e.g. executable path, program
arguments, environment etc.) of a job to be submitted.

Usage example:

  jd = saga.job.Description()

  jd.executable          = /usr/local/bin/blast")
  jd.arguments           = ["-i", "/data/in/x_42"]
  jd.spmd_variation      = "MPI"
  jd.number_of_processed = 42

  js = saga.job.Service ("sge://localhost")
  j  = js.submit_job    (jd)

Class Variable Details

executable

Defines the command to execute.

  • This his is the only required attribute/property.
  • It can be a full pathname, a pathname relative to the working_directory as evaluated on the execution host, or a executable name to be searched in the target host's PATH environment (if available).

Example (using Python properties):

 jd = saga.job.Description()
 jd.executable = "/usr/local/bin/blast"

Example (using the attribute interface):

 jd = saga.job.Description()
 jd.set_attribute("Executable", "/usr/local/bin/blast")
Value:
property(** executable())

arguments

List of arguments for the executable

  • This attribute is optional.
  • The order of arguments is, in general, significant.
  • The individual arguments are not expanded by the user environment (i.e., don't use environment variables like $HOME)

Example (using Python properties):

 jd = saga.job.Description()
 jd.arguments = ["--prefix", "/usr/local/bin"]

Example (using the attribute interface):

 jd = saga.job.Description()
 jd.set_attribute("Arguments". ["--prefix", "/usr/local/bin"])
Value:
property(** arguments())

environment

The environment variables to set in the job's execution context.

  • exported into the job environment
  • format: 'key=value'
  • The individual variable values are not expanded by the user environment (i.e., don't use environment variables like $HOME in values)

Example (using Python properties):

 jd = saga.job.Description()
 jd.environment = ["PATH=/bin:/usr/bin", "TARGET=KingKong"]

Example (using the attribute interface):

 jd = saga.job.Description()
 jd.set_attribute("Environment". ["PATH=/bin:/usr/bin", "TARGET=KingKong"])

wall_time_limit

The hard limit (in minutes) for the total job runtime.

  • The job is not necessarily killed when it overruns the specified time, but no guarantees are made.
  • The value is used by the scheduler to optimized scheduling.

Example (using Python properties):

 jd = saga.job.Description()
 jd.wall_time_limit = "42"

working_directory

The working directory for the job.

  • If the directory does not exist, it will be created.
  • Relative path names (for output/error/file_staging) are relative to this working directory.
  • The backend may choose an arbitrary working directory if none is explicitly specified.

Example (using Python properties):

 jd = saga.job.Description()
 jd.working_directory = "/scratch/cpt_hook/"

Example (using the attribute interface):

 jd = saga.job.Description()
 jd.set_attribute("WorkingDirectory", "/scratch/cpt_hook/")

property(** working_directory())

total_cpu_count

Total number of cpus requested for this job.

  • This attribute is used for jobs spanning more than one process instance, such as for MPI jobs etc.
  • The backend does not make any assumptions about the job distribution over the requested processes, nor does it give any connectivity guarantees (NUMA, shared memory, network interconnects etc.)

Example (using Python properties):

 jd = saga.job.Description()
 jd.cpu_count = 42

Example (using the attribute interface):

 jd = saga.job.Description()
 jd.set_attribute("CpuCount", 42)
Value:
property(** total_cpu_count())

number_of_processes

Number of processes to launch

  • This attribute specifies how many process instances the application is expected to have.
  • Specifying this attribute does *not* automatically create those instances, but instead acts as a hint (boundary condition) to the scheduler.

Example (using Python properties):

 jd = saga.job.Description()
 jd.number_of_processes = 42

Example (using the attribute interface):

 jd = saga.job.Description()
 jd.set_attribute("NumberOfProcesses", 42)
Value:
property(** number_of_processes())

spmd_variation

SPMD job type and startup mechanism (MPI, OpenMP, None, ...)

  • example values: MPI, GridMPI, IntelMPI, LAM-MPI, MPICH1, MPICH2, MPICH-G, OpenMP, PVM
  • 'None' indicates that the application is not a SPMD application.
  • Specifying this value influences the mechanism the backend will use to create the application process instances (fork, mpirun, ...)

Example (using Python properties):

 jd = saga.job.Description()
 jd.spmd_variation = "MPI"

Example (using the attribute interface):

 jd = saga.job.Description()
 jd.set_attribute("SpmdVariation", "MPI")
