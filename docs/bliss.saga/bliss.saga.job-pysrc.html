 4  ''' B{Job Management} 
 5   
 6      The job management API allows to create (via 
 7      a L{job.Service}) and manage L{job.Job}s. A job is, for the purpose 
 8      of the SAGA API, an instance of a specific application, i.e. one or more 
 9      processes of a specific executable.   
10   
11      B{Example}:: 
12   
13       # create a job description... 
14       jd = saga.job.Description() 
15       
16       # ... and fill it with information about the job to be started 
17       jd.executable = "/usr/local/bin/blast" 
18       jd.arguments = ["-i", "/data/in/x_42"] 
19       jd.spmd_variation = "MPI" 
20       jd.number_of_processes = 64 
21       
22       # the job is to be started via globus on some remote host: contact the 
23       # respective submission point via its URL:  
24       js = saga.job.Service("gram://remote.host.net/") 
25   
26       # the thus created job service instance accepts the previously created job 
27       # description and instantiates a job instance. 
28       j = js.create_job(jd) 
29       
30       # a job is a stateful representation of the remote (set of) process(es).  It 
31       # can be run, suspended, resumed and waited upon: 
32       j.run() 
33       j.suspend() 
34       j.resume() 
35       j.wait() 
36   
37       # during the execution, and after termination, the job can be inspected for 
38       # state, and for some additional information 
39       print "id   :  %s"  %  str(j.get_job_id()) 
40       print "state:  %s"  %  str(j.get_state()) 
41       print "descr:  %s"  %  str(j.get_description()) 
42   
43   
44      The above example shows most of the semantics of the job package -- additional 
45      information, such as about file staging and state model etc., can be found in 
46      the individual class documentations. 
47  '''
