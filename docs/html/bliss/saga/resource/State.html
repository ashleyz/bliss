State¶

class bliss.saga.resource.State¶

The State attribute is used to describe the state of resource instances, in particular of instances of bliss.saga.resource.Compute and bliss.saga.resource.Storage.

Active = 2¶

Accepting workload, workload is executed.

Destroyed = 8¶

Closed by user.

Draining = 4¶

workload still executed, not accepting new work items.

Expired = 16¶

Closed by system.

Failed = 32¶

Something went wrong.

Final = 56¶

simplify management of the final states

Pending = 1¶

accepting workload, will eventually become active or fail.

Running = 7¶

simplify management of the non-final states

Unknown = 0¶

wut?
